# Code Style

## General

- Prefer existing local patterns.
- Keep changes scoped to the request.
- Avoid broad refactors unless they are required for the task.
- Add comments only when they explain non-obvious intent.

## Files

- Keep hand-written files small enough to review.
- Split code when a file accumulates unrelated responsibilities.

## Tests

- Add or update tests when behavior changes.
- Use focused tests for narrow fixes.
- Broaden validation when shared behavior or public workflows change.

## Documentation

- Update docs when commands, behavior, runtime, UI, security or operation
  changes.
