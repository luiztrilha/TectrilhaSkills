# Plans

Use plans for work that is broad, risky, user-visible or cross-cutting.

## Plan Format

```markdown
# <Plan Title>

## Goal

## Scope

## Non-Goals

## Files / Areas

## Plan Of Work

- [ ] Step

## Validation

## Decision Log

## Visual Evidence

## Outcome
```

## Rules

- Keep the plan current while work progresses.
- Mark steps complete only after they are actually done.
- Record decisions that change scope or approach.
- For UI work, include screenshot or browser evidence.
- At close, record validation and residual risk.
